## Endpoint

```
POST https://api.cencori.com/api/ai/rag
```

Generate AI responses enhanced with relevant context from your memory namespace. Combines semantic search with chat completions for contextually-aware responses.

## Authentication

Requires API key in `Authorization` header or `CENCORI_API_KEY` header.

## How It Works

1. **Embed Query** - Your question is converted to a vector embedding
2. **Search Memory** - Similar memories are retrieved from the namespace
3. **Inject Context** - Relevant memories are added to the prompt
4. **Generate Response** - AI generates response using the context

## Request Body

<ParamField body="messages" type="array" required>
  Conversation messages (same format as chat endpoint)
</ParamField>

<ParamField body="namespace" type="string" required>
  Memory namespace to search for relevant context
</ParamField>

<ParamField body="model" type="string">
  AI model for generation
  
  Default: Uses project default or `gemini-2.0-flash`
</ParamField>

<ParamField body="temperature" type="number">
  Sampling temperature (0-2)
  
  Default: `0.7`
</ParamField>

<ParamField body="maxTokens" type="number">
  Maximum tokens to generate. Alias: `max_tokens`
</ParamField>

<ParamField body="stream" type="boolean">
  Enable streaming responses
  
  Default: `false`
</ParamField>

<ParamField body="limit" type="number">
  Number of memories to retrieve
  
  Default: `5`
  
  Range: 1-20
</ParamField>

<ParamField body="threshold" type="number">
  Similarity threshold (0-1)
  
  Default: `0.5`
  
  Higher values = more relevant results
</ParamField>

<ParamField body="include_sources" type="boolean">
  Include source memories in response
  
  Default: `true`
</ParamField>

  Generated assistant message

    <ResponseField name="role" type="string">

  </Expandable>

<ResponseField name="sources" type="array">

  <Expandable title="Source">

      Memory content
    </ResponseField>
    <ResponseField name="metadata" type="object">
      Custom metadata attached to memory
    </ResponseField>
    <ResponseField name="similarity" type="number">
      Similarity score (0-1)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="model" type="string">
  Model used for generation
</ResponseField>

<ResponseField name="provider" type="string">
  Provider used
</ResponseField>

<ResponseField name="usage" type="object">
  Token usage statistics
</ResponseField>

<ResponseField name="latency_ms" type="number">
  Total latency including search and generation
</ResponseField>

### Using with SDK

```typescript
// RAG is not directly supported by OpenAI SDK
// Use fetch or axios instead

async function ragQuery(question: string, namespace: string) {
  const response = await fetch('https://api.cencori.com/api/ai/rag', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.CENCORI_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      messages: [{role: 'user', content: question}],
      namespace,
      limit: 5
    })
  });
  
  return await response.json();
}

const result = await ragQuery(
  'How do I reset my password?',
  'faq'
);

console.log(result.message.content);
console.log('Sources used:', result.sources.length);
```

## Best Practices

1. **Optimize memory content**
   - Store concise, focused information
   - Include relevant context in each memory
   - Avoid redundancy

2. **Use meaningful metadata**
   - Add source URLs, dates, categories
   - Helps with filtering and attribution
   - Useful for debugging

3. **Tune retrieval parameters**
   - Start with `limit: 5, threshold: 0.5`
   - Increase limit for complex queries
   - Raise threshold for precision

4. **Organize namespaces**
   - Separate namespaces by domain/topic
   - Easier maintenance and updates
   - Better search relevance

5. **Handle no results**
   - Set appropriate threshold
   - Provide fallback responses
   - Guide users to refine queries

6. **Monitor quality**
   - Review sources returned
   - Track user feedback
   - Iterate on memory content

7. **Update memories regularly**
   - Keep information current
   - Remove outdated content
   - Re-embed when content changes

## Security Features

RAG endpoint includes Cencori's security layers:

- **Input scanning** for prompt injection and jailbreaks
- **Output filtering** for sensitive data leakage
- **PII detection** in queries and responses
- **Security incident logging** for violations

Blocked queries return:

```json
{
  "error": "content_filtered",
  "message": "Your request was blocked by security policy.",
  "reasons": ["prompt_injection"]
}
```

HTTP Status: `403`

## Error Responses

### Missing Namespace

```json
{
  "error": "namespace is required"
}
```

HTTP Status: `400`

### Namespace Not Found

If namespace doesn't exist, query proceeds with empty context (no memories found).

### Provider Not Configured

```json
{
  "error": "Provider 'openai' is not configured",
  "message": "Please add your openai API key in project settings."
}
```

HTTP Status: `400`

## Pricing

RAG costs include:

1. **Embedding generation** - Query embedding ($0.02/1M tokens)
2. **LLM generation** - Response generation (varies by model)

Example with GPT-4o:
- Query embedding: ~$0.000001
- Generation (500 tokens): ~$0.015
- Total: ~$0.015 per query

Use cheaper models like `gemini-2.0-flash` to reduce costs.

## Comparison: RAG vs Manual

### Manual Approach

```typescript
// 1. Generate embedding
const embedding = await client.embeddings.create({
  input: query,
  model: 'text-embedding-3-small'
});

// 2. Search vector database
const results = await vectorDB.search(embedding.data[0].embedding);

// 3. Format context
const context = results.map(r => r.content).join('\n');

// 4. Generate response
const completion = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    {role: 'system', content: `Context: ${context}`},
    {role: 'user', content: query}
  ]
});
```

### Using RAG Endpoint

```typescript
// All-in-one
const response = await ragQuery(query, 'namespace');
```

Benefits:
- Single API call
- Built-in security scanning
- Automatic logging
- Optimized performance
- Cost tracking
