# Secrets Detection

Cencori Scan detects 30+ types of hardcoded API keys, tokens, and secrets using advanced regex patterns.

## AI Provider Keys

### OpenAI

<ResponseField name="OpenAI API Key" severity="critical">
  **Pattern**: `sk-[a-zA-Z0-9]{20}T3BlbkFJ[a-zA-Z0-9]{20}`
  
  **Example**: `sk-abc123...T3BlbkFJ...xyz789`
  
  Detects legacy OpenAI API keys.
</ResponseField>

<ResponseField name="OpenAI Project Key" severity="critical">
  **Pattern**: `sk-proj-[a-zA-Z0-9_-]{80,}`
  
  **Example**: `sk-proj-abc123def456...`
  
  Detects new project-scoped OpenAI keys.
</ResponseField>

### Anthropic

<ResponseField name="Anthropic API Key" severity="critical">
  **Pattern**: `sk-ant-[a-zA-Z0-9-]{90,}`
  
  **Example**: `sk-ant-api03-abc123...`
  
  Detects Claude/Anthropic API keys.
</ResponseField>

### Google AI

<ResponseField name="Google API Key" severity="critical">
  **Pattern**: `AIza[0-9A-Za-z_-]{35}`
  
  **Example**: `AIzaSyAbc123...`
  
  Detects Google Cloud AI and Maps API keys.
</ResponseField>

### Replicate

<ResponseField name="Replicate API Token" severity="critical">
  **Pattern**: `r8_[a-zA-Z0-9]{38}`
  
  **Example**: `r8_abc123def456...`
</ResponseField>

### Hugging Face

<ResponseField name="Hugging Face Token" severity="critical">
  **Pattern**: `hf_[a-zA-Z0-9]{34}`
  
  **Example**: `hf_abc123def456...`
</ResponseField>

## Cloud & Infrastructure

### AWS

<ResponseField name="AWS Access Key ID" severity="critical">
  **Pattern**: `AKIA[0-9A-Z]{16}`
  
  **Example**: `AKIAIOSFODNN7EXAMPLE`
</ResponseField>

<ResponseField name="AWS Secret Access Key" severity="critical">
  **Pattern**: `aws_secret_access_key\s*[:=]\s*["'][A-Za-z0-9/+=]{40}["']`
  
  **Example**: `aws_secret_access_key = "wJalrXUtnFEMI/K7MDENG/..."`
</ResponseField>

### Supabase

<ResponseField name="Supabase Service Role Key" severity="critical">
  **Pattern**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9\.[a-zA-Z0-9_-]+\.[a-zA-Z0-9_-]+`
  
  **Example**: JWT token starting with `eyJhbGci...`
  
  Detects Supabase service role keys (JWT format).
</ResponseField>

<ResponseField name="Supabase Anon Key" severity="medium">
  **Pattern**: `SUPABASE_ANON_KEY\s*[:=]\s*["']eyJ[^"']+["']`
  
  Only flagged if hardcoded (anon keys are public by design).
</ResponseField>

### Firebase

<ResponseField name="Firebase Database URL" severity="medium">
  **Pattern**: `https://[a-z0-9-]+\.firebaseio\.com`
  
  **Example**: `https://my-app.firebaseio.com`
</ResponseField>

## Database Connections

### MongoDB

<ResponseField name="MongoDB Connection String" severity="critical">
  **Pattern**: `mongodb(\+srv)?://[^@\s]+@[^\s"']+`
  
  **Example**: `mongodb+srv://user:pass@cluster.mongodb.net/db`
  
  Detects connection strings with embedded credentials.
</ResponseField>

### PostgreSQL

<ResponseField name="PostgreSQL Connection String" severity="critical">
  **Pattern**: `postgres(ql)?://[^\s"']+`
  
  **Example**: `postgresql://user:pass@localhost:5432/db`
</ResponseField>

### MySQL

<ResponseField name="MySQL Connection String" severity="critical">
  **Pattern**: `mysql://[^\s"']+`
  
  **Example**: `mysql://user:pass@localhost:3306/db`
</ResponseField>

### Redis

<ResponseField name="Redis Connection String" severity="high">
  **Pattern**: `redis://[^\s"']+`
  
  **Example**: `redis://user:pass@localhost:6379`
</ResponseField>

## Generic Patterns

### Private Keys

<ResponseField name="Private Key" severity="critical">
  **Pattern**: `-----BEGIN (RSA |EC |DSA |OPENSSH )?PRIVATE KEY-----`
  
  Detects private keys in PEM format:
  - RSA keys
  - EC keys
  - DSA keys
  - OpenSSH keys
</ResponseField>

### Generic API Keys

<ResponseField name="Generic API Key Assignment" severity="high">
  **Pattern**: `(api_key|apikey|api_secret|secret_key)\s*[:=]\s*["'][a-zA-Z0-9_-]{20,}["']`
  
  **Example**: `api_key = "abc123def456..."`
  
  Catches generic API key assignments.
</ResponseField>

### Passwords

<ResponseField name="Password Assignment" severity="high">
  **Pattern**: `(password|passwd|pwd)\s*[:=]\s*["'][^"']{8,}["']`
  
  **Example**: `password = "mySecretPass123"`
  
  Detects hardcoded passwords.
</ResponseField>

### JWT Secrets

<ResponseField name="JWT Secret Assignment" severity="critical">
  **Pattern**: `JWT_SECRET\s*[:=]\s*["'][^"']{16,}["']`
  
  **Example**: `JWT_SECRET = "my-super-secret-key"`
</ResponseField>

<ResponseField name="Hardcoded JWT Sign" severity="critical">
  **Pattern**: `jwt\.(sign|verify)\s*\([^,]+,\s*["'][^"']{10,}["']`
  
  **Example**: `jwt.sign(payload, "hardcoded-secret")`
</ResponseField>

### OAuth

<ResponseField name="OAuth Client Secret" severity="critical">
  **Pattern**: `client_secret\s*[:=]\s*["'][a-zA-Z0-9_-]{20,}["']`
  
  **Example**: `client_secret = "abc123..."`
</ResponseField>

<ResponseField name="Google Client Secret" severity="critical">
  **Pattern**: `GOOGLE_CLIENT_SECRET\s*[:=]\s*["'][^"']+["']`
  
  **Example**: `GOOGLE_CLIENT_SECRET = "GOCSPX-..."`
</ResponseField>

## How Detection Works

### 1. File Scanning

The scanner:
1. Reads all files in your project
2. Skips ignored directories (`node_modules`, `.git`, etc.)
3. Scans scannable extensions (`.js`, `.ts`, `.py`, `.env`, etc.)

### 2. Pattern Matching

Each pattern uses regex to match:
- Specific key formats (e.g., `sk-proj-...`)
- Common variable assignments
- Connection strings with credentials

### 3. False Positive Filtering

The scanner automatically filters:
- Test files (`.test.js`, `.spec.ts`)
- Documentation (`.md` files)
- Example domains (`example.com`)
- Placeholder values

### 4. Redaction

Detected secrets are redacted in output:

```
sk-proj-**** (first 4 and last 4 chars shown)
```

## Severity Levels

<ResponseField name="Critical" type="F-Tier">
  **Secrets that provide full access:**
  - Production API keys
  - Database credentials
  - Private keys
  - Service role tokens
</ResponseField>

<ResponseField name="High" type="D-Tier">
  **Secrets that provide significant access:**
  - Bot tokens
  - Webhook secrets
  - API secrets
</ResponseField>

<ResponseField name="Medium" type="C-Tier">
  **Secrets with limited scope:**
  - Test API keys
  - Public-ish tokens (Supabase anon keys)
  - Database URLs without passwords
</ResponseField>

## What's NOT Detected

<Warning>
  The scanner cannot detect:
  - Encrypted or encoded secrets
  - Secrets stored in binary files
  - Secrets dynamically constructed at runtime
  - Custom API key formats not in the pattern list
</Warning>

## Best Practices

<Tip>
  **Use environment variables** for all secrets:
  
  ```typescript
  const apiKey = process.env.OPENAI_API_KEY;
  ```
</Tip>

<Tip>
  **Never commit** `.env` files. Add to `.gitignore`:
  
  ```gitignore
  .env
  .env.local
  .env*.local
  ```
</Tip>

<Tip>
  **Use `.env.example`** for documentation:
  
  ```bash
  OPENAI_API_KEY=your_key_here
  ```
</Tip>
