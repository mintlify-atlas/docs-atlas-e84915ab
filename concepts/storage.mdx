## Semantic caching

Cache AI responses based on semantic similarity to reduce costs and latency.

### How it works

1. Hash the prompt and generate an embedding
2. Search for similar cached responses (cosine similarity > 0.95)
3. If found, return cached response
4. If not found, call AI provider and cache the result

```typescript
// Coming soon
const response = await cencori.ai.chat({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'What is the capital of France?' }],
  cache: {
    enabled: true,
    ttl: 86400, // 24 hours
    similarityThreshold: 0.95
  }
});

if (response.cached) {
  console.log('Cache hit! Saved $0.002');
}
```

## Database architecture

Cencori Storage is built on PostgreSQL with Supabase:

- **Database**: PostgreSQL 15 with pgvector extension
- **Auth**: Supabase Auth with Row Level Security (RLS)
- **Storage**: Supabase Storage for file uploads
- **Realtime**: Supabase Realtime for live updates

### Vector indexes

```sql
-- HNSW index for fast approximate search
CREATE INDEX vectors_embedding_idx 
ON vectors 
USING hnsw (embedding vector_cosine_ops);

-- Metadata index for filtering
CREATE INDEX vectors_metadata_idx 
ON vectors 
USING GIN (metadata);
```
