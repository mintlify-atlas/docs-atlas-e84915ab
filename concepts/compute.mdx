## Use cases

### AI agents

Run autonomous agents that combine LLM calls with custom logic:

```python
# Coming soon
from cencori import Cencori

cencori = Cencori(api_key='your-api-key')

result = await cencori.compute.run(
  runtime='python',
  code="""
import requests
from cencori import ai

# Fetch data from API
data = requests.get('https://api.example.com/data').json()

# Analyze with AI
response = ai.chat(
  model='gpt-4o',
  messages=[{
    'role': 'user',
    'content': f'Summarize this data: {data}'
  }]
)

return response.content
  """
)

print(result.output)
```

### Tool execution

Execute tools returned by AI models securely:

```typescript
// Coming soon
const response = await cencori.ai.chat({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'What is the weather in SF?' }],
  tools: [{
    type: 'function',
    function: {
      name: 'get_weather',
      description: 'Get current weather for a location',
      parameters: {
        type: 'object',
        properties: {
          location: { type: 'string' }
        }
      }
    }
  }]
});

if (response.toolCalls) {
  for (const toolCall of response.toolCalls) {
    const result = await cencori.compute.executeTool({
      name: toolCall.function.name,
      arguments: JSON.parse(toolCall.function.arguments)
    });
  }
}
```

## Security

### Sandboxed execution

All code runs in isolated containers with:

- **Resource limits**: CPU, memory, and execution time caps
- **Network isolation**: Controlled outbound access
- **File system isolation**: Read-only file system by default
- **Secrets injection**: Secure access to environment variables

### Runtime policies

Define policies to control what your code can do:

```typescript
// Coming soon
const result = await cencori.compute.run({
  runtime: 'python',
  code: myCode,
  policy: {
    allowedDomains: ['api.example.com'],
    maxMemoryMB: 512,
    maxDurationSec: 30,
    allowNetworkAccess: true,
    allowFileWrites: false
  }
});
```

## Performance

### Cold start optimization

Compute instances are pre-warmed to minimize cold start latency:

- **< 100ms** for Node.js functions
- **< 500ms** for Python functions
- **< 2s** for container workloads

### Auto-scaling

Automatic scaling based on request volume:

```
┌──────────────────────────────────────────────────┐
│  Request Volume        │  Instances             │
├──────────────────────────────────────────────────┤
│  0-100 req/s           │  1-5                   │
│  100-1000 req/s        │  5-50                  │
│  1000-10000 req/s      │  50-500                │
│  10000+ req/s          │  500+ (contact sales)  │
└──────────────────────────────────────────────────┘
```
