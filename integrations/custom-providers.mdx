# Custom Provider Configuration

Cencori supports custom AI providers that implement OpenAI-compatible or Anthropic-compatible APIs. This allows you to use local models, private deployments, or alternative providers through Cencori's gateway.

## How It Works

Custom providers work by:

1. Accepting a base URL for your provider's API
2. Formatting requests in either OpenAI or Anthropic format
3. Routing requests through Cencori for observability and security
4. Normalizing responses to Cencori's unified format

## Adding a Custom Provider

<Steps>
  <Step title="Get your provider details">
    Collect the following information:
    
    - **Base URL**: The API endpoint (e.g., `https://api.example.com`)
    - **API Key**: Your provider's API key (optional)
    - **Format**: Whether it's OpenAI-compatible or Anthropic-compatible
    - **Model IDs**: The model identifiers your provider supports
  </Step>

  <Step title="Add provider to Cencori">
    In your [Cencori Dashboard](https://cencori.com/dashboard):
    
    1. Go to **Settings** > **Providers**
    2. Click **Add Provider**
    3. Select **Custom Provider**
    4. Fill in the details:
       - Name: A friendly name for your provider
       - Base URL: Your provider's API endpoint
       - API Key: Your authentication key (if required)
       - Format: `openai` or `anthropic`
    5. Click **Test Connection**
    6. Click **Save**
  </Step>

  <Step title="Use your custom provider">
    Use your custom provider like any other:

    ```typescript
    const response = await cencori.ai.chat({
      model: 'your-custom-model',
      messages: [{ role: 'user', content: 'Hello!' }]
    });
    ```
  </Step>
</Steps>

## Supported Provider Types

### OpenAI-Compatible Providers

These providers implement the OpenAI Chat Completions API:

- **Local Models**: Ollama, LM Studio, LocalAI
- **Cloud Providers**: Together AI, Groq, Anyscale
- **Custom Deployments**: Self-hosted vLLM, FastAPI wrappers
- **Azure OpenAI**: Microsoft's hosted OpenAI models

### Anthropic-Compatible Providers

Providers that implement Anthropic's Messages API:

- **AWS Bedrock**: Claude models on AWS
- **Custom Deployments**: Self-hosted Claude-compatible APIs

## Usage Examples

### Using the SDK

<CodeGroup>
```typescript TypeScript
import { Cencori } from 'cencori';

const cencori = new Cencori({
  apiKey: process.env.CENCORI_API_KEY
});

// Use your custom provider
const response = await cencori.ai.chat({
  model: 'your-custom-model',
  messages: [{ role: 'user', content: 'Hello!' }]
});

console.log(response.content);
```

```python Python
from cencori import Cencori

cencori = Cencori(api_key="csk_your_key")

response = cencori.ai.chat(
    model="your-custom-model",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.content)
```
</CodeGroup>

### With Vercel AI SDK

```typescript
import { cencori } from 'cencori/vercel';
import { streamText } from 'ai';

const result = await streamText({
  model: cencori('your-custom-model'),
  messages: [{ role: 'user', content: 'Hello!' }]
});
```

## Advanced Configuration

### Custom Headers

Add custom headers for authentication or routing:

```typescript
// Configure in Dashboard:
Headers:
  X-Custom-Header: value
  X-API-Version: v1
```

### Pricing Configuration

Set custom pricing for cost tracking:

```typescript
// In Dashboard:
Input Cost: $0.001 per 1K tokens
Output Cost: $0.002 per 1K tokens
```

### Timeout Configuration

Set request timeout (default: 30s):

```typescript
const response = await cencori.ai.chat({
  model: 'your-custom-model',
  messages: [{ role: 'user', content: 'Hello!' }],
  timeout: 60000  // 60 seconds
});
```

## Troubleshooting

### Connection Failed

If connection tests fail:

1. Verify the base URL is correct and accessible
2. Check that the API key is valid (if required)
3. Ensure your provider implements the correct API format
4. Check firewall/network settings if using local providers

### Incompatible Format

If you see format errors:

1. Verify you selected the correct format (OpenAI vs Anthropic)
2. Check your provider's API documentation
3. Test the provider directly with curl to verify responses

### Model Not Found

If the model isn't recognized:

1. Verify the model ID is correct for your provider
2. Check that the model is available/loaded (for local providers)
3. Consult your provider's model list

## Security Considerations

### Local Providers

- Use `http://localhost` or `http://127.0.0.1` for local-only access
- Don't expose local servers to the internet
- Use API keys even for local providers when possible

### Cloud Providers

- Always use HTTPS for base URLs
- Rotate API keys regularly
- Use environment variables for sensitive data
- Monitor usage in both Cencori and provider dashboards
